# 京大院試ドリルアプリ（app-kyoto）Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `app/` を複製した京大院試用ドリル `app-kyoto/` を作り、和訳・和文英訳・補充四択・論述(AI採点)の4モードで過去問を反復演習できるようにする。

**Architecture:** 既存 `app/index.html`（1068行・単一HTMLのPWA）を土台に複製し、ストレージキーを分離する。既存の和訳(`type:"tr"`)＋Claude採点＋SRS＋Gist同期エンジンを流用し、`type:"en"`（和文英訳）と `type:"essay"`（論述）の2モードを追加する。問題は `app-kyoto/questions.json` に投入し、node製の `validate.mjs` でスキーマ検証する。

**Tech Stack:** 単一HTML + バニラJS（ビルド・フレームワーク無し）、Anthropic Messages API（ブラウザ直叩き, `claude-opus-4-8`）、node（検証スクリプト用のみ）、PWA(manifest)。

## Global Constraints

- スマホ前提UI。既存 `app/` のCSS・レイアウト・操作系を踏襲する（新規CSSは最小限）。
- 問題の追加・編集は必ず `app-kyoto/questions.json` 経由。`index.html` に問題を直書きしない。
- ストレージキーは `app/`（`kdai_v2` 等）と衝突させない。`app-kyoto` は `kdai_kyoto_*` 系を使う。
- JSのテストフレームワークは導入しない（既存コードにテストは無い）。データは `validate.mjs`（node）で自動検証、UI挙動はブラウザで確認する。
- 既存の外部通信先（`https://api.anthropic.com/v1/messages`）・モデルID `claude-opus-4-8` は変更しない。
- 和訳の採点は10点満点・8点以上で◯（既存挙動）。和文英訳も同基準に合わせる。
- コミットは日本語メッセージ、末尾に `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`。

- [ ] **Step 2: 検証スクリプト `validate.mjs` を作成**

Create `app-kyoto/validate.mjs`:

```js
import { readFileSync } from 'node:fs';

const path = new URL('./questions.json', import.meta.url);
let data;
try {
  data = JSON.parse(readFileSync(path, 'utf8'));
} catch (e) {
  console.error('JSON parse error:', e.message);
  process.exit(1);
}

const errs = [];
if (!Array.isArray(data)) errs.push('top-level は配列である必要があります');

const ids = new Set();
const TYPES = new Set([undefined, 'tr', 'en', 'essay']);
data.forEach((x, i) => {
  const at = `#${i}(id=${x && x.id})`;
  if (typeof x !== 'object' || x === null) { errs.push(`${at}: オブジェクトでない`); return; }
  if (typeof x.id !== 'number') errs.push(`${at}: id は数値`);
  if (ids.has(x.id)) errs.push(`${at}: id 重複`); else ids.add(x.id);
  for (const k of ['part', 'cat', 'q', 'a', 'e']) {
    if (typeof x[k] !== 'string' || !x[k].trim()) errs.push(`${at}: ${k} は非空文字列`);
  }
  if (!TYPES.has(x.type)) errs.push(`${at}: 未知の type=${x.type}`);
  if (x.type === undefined) {
    if (!Array.isArray(x.d) || x.d.length < 1) errs.push(`${at}: 四択には d(誤答配列, 長さ≥1) が必要`);
    else if (x.d.some(o => typeof o !== 'string' || !o.trim())) errs.push(`${at}: d の要素は非空文字列`);
    if (typeof x.a === 'string' && Array.isArray(x.d) && x.d.includes(x.a)) errs.push(`${at}: 正解 a が誤答 d に混入`);
  }
  if (x.type === 'essay') {
    if (x.kw !== undefined && (!Array.isArray(x.kw) || x.kw.some(w => typeof w !== 'string'))) errs.push(`${at}: kw は文字列配列`);
    if (x.limit !== undefined && typeof x.limit !== 'number') errs.push(`${at}: limit は数値`);
  }
  if (x.hint !== undefined && typeof x.hint !== 'string') errs.push(`${at}: hint は文字列`);
});

if (errs.length) { console.error('検証NG:\n' + errs.map(e => ' - ' + e).join('\n')); process.exit(1); }
console.log(`OK: ${data.length}件`);
```
