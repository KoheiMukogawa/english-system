# 航大 気象・環境ドリル 設計書

- 作成日: 2026-07-07
- 対象試験: 航空大学校 入学第一次試験 総合Ⅱ の「気象・環境」分野
- 位置づけ: 既存の英語ドリル（`app/`）の姉妹アプリ

## 1. 背景と目的

既存の英語ドリル（Leitner式SRS＋4択＋gist同期＋スマホ前提UI）が安定稼働している。
同じ航大入試の「気象・環境」分野を、同じ学習体験でドリル化する。

### 出題範囲（リサーチ結果）

令和9年度（2027年入試）から筆記の出題範囲が明示・限定される:

- 地学基礎「**地球の環境**」のみ（＝環境問題）
- 地学「**大気の構造と運動**」のみ（＝気象）
- 時事問題・社会常識は廃止、化学・生物は出題されない

本ドリルの対象「気象＋環境」はこの新公式範囲に一致する（旧範囲の受験でも有効）。

**前提**: 受験年度は令和9年度（2027）以降の新範囲を主眼に置く。旧範囲でも同じテーマで対応可能なため、この前提が外れても設計変更は不要。

### 出題テーマ（過去問傾向）

- **気象（大気の構造と運動）**: 大気圏の鉛直構造 / 顕熱・潜熱 / 気温減率（乾燥・湿潤断熱） / 雲の発生 / 前線（4種＋前線記号）📊 / 低気圧・高気圧・南岸低気圧📊 / 台風 / フェーン現象 / 高層天気図（気圧の谷・尾根・偏西風）📊 / 地上天気図の読み取り📊 / 衛星可視画像📊
- **環境（地球の環境）**: 地球温暖化★ / オゾンホール★ / 酸性雨 / PM2.5 / ヒートアイランド / カーボンサイクル（＋エルニーニョ等の余地）
- 📊＝図の読み取りを伴う。★＝頻出・再出題あり。
- 傾向: 出題は狭く固定的で、一度出た問題が再出題される → 少数精鋭の高頻度バンクが最適。

## 2. アーキテクチャ

**方針: 既存アプリの複製（別アプリ）**

- `app/` を `app-met/` として複製し、英語専用機能を削って気象・環境用に改造する。
- 安定稼働中の英語アプリ（`app/`）には一切手を加えない。

### 残す機能（英語版から流用）

- 4択出題エンジン（`renderQ` / `choose`）
- Leitner SRS（box/due、STATS、スマート復習、復習ボックスに戻す、卒業）
- gist同期
- 自分メモ、問題一覧からの単問演習、履歴消去、PWA(manifest/icons)
- 語源(ety)欄 → 気象でも「関連づけメモ」として有用なため残す（任意フィールド）
- スマホ前提UI、v2.11.0で入れた「PCで入力ボックス自動フォーカス」「Ctrl+Enterショートカット」の思想（該当UIがあれば踏襲）

### 削る機能（英語専用）

- 和訳問題（`type:'tr'`、`renderTr` / `revealTr` / `gradeTr` / `applyTrScore` / TRLOG 一式）
- Claude採点（AI設定、`aiGrade`、APIキー保存、関連メニュー・説明）
- 和訳系メニュー（スマート復習の和訳枠、レベル1/2、間違えた和訳 など）

### 進捗の分離

英語版と進捗・同期が混ざらないよう、キー/ファイル名を分ける:

| 項目 | 英語版 | 気象・環境版 |
|------|--------|--------------|
| localStorage (進捗) | `kdai_v2` | `kdai_met_v1` |
| 同期設定キー | `kdai_sync_v1` | `kdai_met_sync_v1` |
| gistファイル名 | `kdai-progress.json` | `kdai-met-progress.json` |
| 問題データ | `app/questions.json` | `app-met/questions.json` |

## 3. データモデル

`app-met/questions.json` は既存4択スキーマを流用し、画像フィールドを1つ追加する。

```
{
  "id": <数値>,
  "cat": "気象・大気の構造" | "気象・天気図" | "環境" など,
  "q":  "問題文",
  "a":  "正解の選択肢（文字列）",
  "d":  ["誤答1", "誤答2", "誤答3"],
  "e":  "解説",
  "ety": "（任意）関連づけ・語源・覚え方",
  "img": "（任意）img/xxx.svg への相対パス、または軽量インラインSVG文字列"
}
```

- `type` は使わない（全て4択）。和訳type は本アプリに存在しない。
- `img` があるときだけ問題文の下に画像を表示する（`renderQ` の最小変更）。

## 4. 画像方針（著作権セーフ）

- 前線記号・気圧の谷/尾根・大気圏の鉛直構造・断熱変化などの構造図は **自作SVG模式図**（軽量・著作権フリー）。
- 天気図・衛星可視画像の読み取りは **気象庁の公開素材（政府作成物）** をローカル保存、または模式化して自作。
- **実際の過去問の図は使用しない**（以前の過去問PDF削除方針を踏襲）。
- 画像は `app-met/img/` に配置し、`img` フィールドから相対パス参照。SVG推奨（軽量・拡大に強い）。

## 5. コンポーネント構成（app-met/index.html）

英語版の単一HTMLファイル構成を踏襲。主な変更点のみ:

1. `renderQ`: `x.img` があれば問題カードの下に画像ブロックを追加表示。
2. ストリップ: 和訳・Claude採点・語源以外のAI関連の関数とメニュー項目を削除。
3. メニュー: 「気象」「環境」カテゴリ別スタート、スマート復習、復習ボックス、間違えた問題、問題一覧に再構成。
4. 同期/保存キーを上表の値に置換。
5. `manifest.webmanifest` / アイコン / タイトルを気象・環境版に更新。

## 6. 段取り（実装順）

1. **エンジン確定**: `app-met/` 複製 → 英語機能ストリップ → 同期キー分離 → `img` 表示対応。動作確認。
2. **知識問題バンク**: 図なしの知識系（環境の全テーマ＋気象の知識系）を `questions.json` に作成。
3. **図問題バンク**: 自作SVGの図問題を追加（前線記号 → 天気図/高層天気図/衛星画像 読み取りの順）。

各段階で単体で動作確認できる状態を保つ。

## 7. テスト・検証方針

- ブラウザで `app-met/index.html` を開き、出題→回答→SRS記録→次へ→結果 の一連が動くことを確認。
- `img` あり/なし両方の問題が正しく表示されること。
- 進捗が英語版（`kdai_v2`）と混ざらないこと（別localStorageキー）。
- gist同期を設定した場合、英語版の進捗を壊さないこと（別ファイル名）。

## 8. 非目標（YAGNI）

- 計算問題の数値入力・途中式採点（現行の気象出題は4択中心のため不要）。
- 時事・社会常識・法律（新範囲で廃止）。
- 物理・数学（別分野。本アプリの対象外）。
- 英語版とのコード共通化リファクタ（複製で十分、既存を壊すリスクを避ける）。
