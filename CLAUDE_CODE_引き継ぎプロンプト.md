# 引き継ぎプロンプト — 航空大学校 英語「大問I」学習アプリ（PWA化）

> **使い方**：新しいフォルダを作り、その中に次の2ファイルを置いてください →
> ① `questions.json`（全171問データ）② `航大英語_大問I_ドリルアプリ.html`（動くプロトタイプ）。
> そのフォルダで Claude Code を開き、**この文章全体を貼り付けて**ください。

## 背景・目的
- 私は航空大学校（パイロット養成）一次試験を受ける受験生です。英語はCBT形式で、試験日は **2026年7月12日**。
- 対策したいのは英語リーディングの **「大問I（短文空所補充・語彙/文法/イディオム）」**。1文＋4択の形式です。
- 通学・移動中にスマホでタップ学習したい。最重要は次の3点：
  1. **オフラインで動く**（電車内でも全機能）
  2. **進捗が絶対に消えない**（再起動・再訪でも保持、端末移行もできる）
  3. **間違えた問題を優先して復習できる**（苦手だけ自動で出る）
- すでに単一HTMLの**動くプロトタイプ**があります。これを、**インストール可能なPWA**にして無料の静的ホスティングにデプロイしたい。将来は和訳（大問III）・リスニング・語彙ドリルも追加予定。

## いま手元にあるもの（このフォルダ）
1. **`questions.json`** … 全171問のデータ。**これが唯一の正データ源。内容は改変しないこと。**
2. **`航大英語_大問I_ドリルアプリ.html`** … 動作するプロトタイプ。**「求める挙動の仕様書」として必ず読むこと**（学習モード、即時採点、解説表示、簡易SRS、統計トップ画面が実装済み）。同じ171問が埋め込まれている。

## データスキーマ（questions.json：オブジェクトの配列）
```json
{
  "id": 1,
  "part": "Part 1",
  "cat": "前置詞コロケーション",
  "q": "The captain was anxious ( ) the deteriorating weather.",
  "a": "about",
  "d": ["on", "to", "for"],
  "e": "be anxious **about**＝〜を心配して…\n◎ 関連表現…"
}
```
- `q` … 問題文。`( )` が空所。
- `a` … 正解（1つ）。`d` … 誤答3つ。**表示時は `[a, ...d]` を毎回シャッフル**し、位置で正解を覚えられないようにする。
- `e` … 解説。`\n`＝改行、`**…**`＝太字、記号の意味は `✗`＝誤答分析／`◎`＝関連表現／`★`＝航空・気象・医療・時事など頻出テーマ。表示時に軽くMarkdown変換（太字・改行）する。
- パート構成（計171問）：
  - Part 1 前置詞コロケーション（30）
  - Part 2 句動詞（22）
  - Part 3 イディオム・慣用（24）
  - Part 4 文法・語法（35、`cat`に 仮定法/関係詞/比較/時制/第4文型/動名詞・不定詞/不可算・数量/倒置 等）
  - Part 5 ことわざ（10）
  - Part 6 トピック語彙（50、`cat`に ✈航空/☁気象/⚕医療/🏛時事/🔬科学/💼ビジネス/📚一般難語）

## MVP（最優先で作るもの）
1. **PWA化**：Web App Manifest ＋ Service Worker で完全オフライン（初回読込後は機内モードでも全機能）。iPhoneで「ホーム画面に追加」→アイコンから全画面起動。アイコン／テーマカラー／スプラッシュを用意。
2. **進捗の永続化**：**IndexedDB 推奨**（localStorage でも可だが消えにくさ重視）。アプリ再起動・再訪でも保持。加えて **進捗のエクスポート／インポート（JSONファイル or コピー用コード）** を実装し、機種変更・バックアップに対応。
3. **学習モード**：
   - **スマート復習**：優先度順（① 間違えて未マスター → ② 未学習 → ③ 復習途中 → ④ マスター済み）で1回20問。
   - **間違えた問題だけ**：直近で誤答し、まだ再習得していない問題のみ。
   - **パート別** ／ **全問ランダム**。
4. **出題フロー**：1問ずつ表示。選択肢は毎回シャッフル。タップで即採点（正解=緑／選んだ誤答=赤、正解も強調）、`e` を整形して解説表示、「次へ」。セッション終了で結果画面（正答率、間違えた問題をすぐ復習）。
5. **間隔反復（SRS）**：プロトタイプは「2連続正解でマスター」の簡易版。これを **Leitner か SM-2 lite** に格上げ。各問に box/interval/due を持たせ、正解で次回間隔を延長・誤答で短縮し、**due が近い/超過している順**に出題。
6. **統計**：復習待ち／未学習／マスター／累計正答率。連続学習日数（streak）もあると良い。
7. **UI/UX**：モバイルファースト、タップ領域 ≥44px、日本語UI、ダーク/ライト自動、動作軽快。**ランタイムの外部通信ゼロ**（完全オフライン）。

## 推奨技術（最終判断は任せる）
- とにかく **簡単＆無料デプロイ** 重視。
  - 案A：素の HTML/CSS/JS ＋ manifest ＋ Service Worker（ビルド不要・最速）。
  - 案B（推奨）：**Vite + React + TypeScript + Tailwind + `vite-plugin-pwa`**（保守性◎、SW自動生成）。
- データは静的JSONのみ → **バックエンド不要**。**GitHub Pages / Netlify / Vercel** の無料枠に静的デプロイ。
- 端末間同期は将来課題（必要になったら軽量バックエンド、または「エクスポートコード」方式）。

## 今後のロードマップ（MVP後・優先度順）
1. **大問III 和訳モジュール**：自由記述（キーボード入力）＋ 模範解答と「**京大方式**（構文/語彙/訳抜け/時制で減点、ブロック単位の部分点）」ルーブリックで自己採点。MCQとは別UX。解答履歴も保存。
2. **語彙ドリル統合**：私が実際に調べた約250語の「単語→意味／意味→単語」認識モードを別バンクとして追加（`vocab.json` 化する）。
3. **リスニング**：航大は共通テスト（センター）形式に近い。音源が要るため当面はプレースホルダ／外部リンク。
4. 毎日の目標・リマインド（iOS PWA の push は制限が大きい点に留意）。

## 完成の定義（Acceptance Criteria）
- [ ] iPhone のホーム画面に追加し、アイコンから全画面起動できる。
- [ ] 初回読込後、**機内モードで全機能が動く**。
- [ ] アプリ再起動後も進捗が残る。**エクスポート→別端末インポート**で移行できる。
- [ ] 「間違えた問題だけ」「スマート復習」で苦手が優先的に出る。
- [ ] 171問すべて読み込め、コンソールエラーなし。選択肢は毎回シャッフルされる。
